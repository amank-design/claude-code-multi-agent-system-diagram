End-to-end automated pipeline from Jira ticket to reviewed, tested, production-ready code

Overview

The Multi-Agent Development System is an internal engineering tool built on Claude Code (Anthropic). It consists of 9 specialized AI agents that work in a pipeline — from reading Jira tickets and Figma designs, to writing code, tests, reviewing, fixing bugs, and updating documentation.

The system automates the repetitive parts of the development workflow so engineers can focus on architecture, product thinking, and quality control rather than implementation details.

⚠️
Note: This system does not replace engineers. It handles 70-80% of implementation work automatically. Engineers are still responsible for reviewing the output, ensuring business logic is correct, and making architectural decisions.

Language & Framework Support
🌐

Works with ANY Stack

This system is not limited to React or TypeScript. The Codebase Understanding Agent (Agent 02) automatically scans and learns any project's language, framework, and patterns before writing a single line of code.

It works with any stack including React, Vue, Angular, Next.js on the frontend, Node.js, Python, Java, Go, Ruby on Rails on the backend, and React Native or Flutter for mobile. The commands and workflow stay exactly the same regardless of the technology used.

One system for every team and every project in the organisation.

Installation & Setup Guide

Step 1 — Install Claude Code

# Check Node.js version (must be v18+)
node --version

# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Login with org API key
claude
# Select option 2: Anthropic Console account
# Paste API key from console.anthropic.com

Step 2 — Set Up API Key Permanently

# Export key for current session
export ANTHROPIC_API_KEY=sk-ant-api03-your-key-here

# Make it permanent across terminal sessions
echo 'export ANTHROPIC_API_KEY=sk-ant-api03-your-key-here' >> ~/.zshrc
source ~/.zshrc

Step 3 — Navigate to Your Project

cd /path/to/your/project

Step 4 — Create Agent Files and Scripts

# Create agents folder and all required files
mkdir agents
touch run.sh fix.sh review-pr.sh docs.sh review-report.md
touch agents/01-task-breakdown.md agents/02-codebase.md agents/02-codebase-update.md
touch agents/03-figma.md agents/04-dev.md agents/05-test.md
touch agents/06-reviewer.md agents/07-bugfix.md agents/08-docs.md agents/pr-reviewer.md

# Make scripts executable (Mac/Linux)
chmod +x run.sh fix.sh review-pr.sh docs.sh

Step 5 — Create .env File

# Create .env in project root
touch .env

# Add the following content to .env
ANTHROPIC_API_KEY=sk-ant-api03-your-key-here
JIRA_URL=https://yourcompany.atlassian.net
JIRA_EMAIL=you@yourcompany.com
JIRA_API_TOKEN=your-jira-token-here
FIGMA_ACCESS_TOKEN=your-figma-token-here
GITHUB_TOKEN=your-github-token-here

Step 6 — Set Up MCP Connections

# Add Figma Desktop MCP (open Figma app first)
claude mcp add --scope user --transport http figma-desktop http://127.0.0.1:3845/mcp

# Add Atlassian MCP (Jira)
claude mcp add --transport sse atlassian https://mcp.atlassian.com/v1/sse

# Add GitHub MCP
claude mcp add --scope user --transport http github https://api.githubcopilot.com/mcp

# Verify all three are connected
claude mcp list
figma-desktop: Connected
atlassian: Connected
github: Connected

Step 7 — Set Permissions

Create .claude/settings.local.json in your project root:

mkdir -p .claude
# Create settings file with full permissions
{
  "permissions": {
    "allow": [
      "Bash(*)",
      "Write(*)",
      "Read(*)",
      "WebFetch(*)"
    ]
  }
}

Step 8 — Run Your First Ticket

./run.sh BOD-123
# First run will automatically generate codebase-context.md
# This takes 3-10 minutes depending on project size
# All subsequent runs skip this step and are much faster
💡
First Run Note: The first time you run the script, Agent 02 will scan your entire codebase (2,101+ files for this project). This takes 5-10 minutes. After that it only updates incrementally and takes seconds.

Frequently Asked Questions

Can I run multiple tickets at the same time?

Not recommended in the same project folder as agents share context files like subtasks.json and review-report.md. Run one ticket at a time per project. You can run different tickets in different project clones simultaneously.

What happens if the API key expires mid-run?

The pipeline will stop with a credit or auth error. Simply run the same command again after refreshing your key. The codebase-context.md is already saved so it will not re-scan the whole project.

Can I modify the agent prompts?

Yes. All agent prompts are in the agents/ folder as plain markdown files. Edit them to match your team's specific conventions or review criteria. Changes apply immediately on the next run.

Does it work on Windows?

Yes. Use .bat scripts instead of .sh scripts. The logic is identical but uses Windows batch syntax. Run in PowerShell or Command Prompt.

What if there is no Figma design for a ticket?

Agent 03 checks for figma-url.txt. If it contains NO_FIGMA_LINK or does not exist, Agent 03 writes NO_FIGMA_DESIGN to figma-specs.md and skips automatically. The rest of the pipeline continues normally.

How do I update the codebase context manually?

Delete codebase-context.md and run any script. It will automatically trigger a full rescan. Or make 20+ changes and run a script — the auto-detection will trigger a full rescan for you.

Can this system handle backend and frontend in the same monorepo?

Yes. Agent 02 scans the full project and understands multiple languages and frameworks in the same repository. Agent 04 will write code in the correct language for each part of the codebase based on what it reads in codebase-context.md.
